# Адміністрування бота

Цей документ для людини, яка обслуговує бота — без глибоких знань коду.

---

## Що вміє бот

- Надсилає заплановані оголошення в групові чати за розкладом (cron)
- Надсилає особисті повідомлення обраним користувачам
- Відповідає на команди: `/start`, `/help`, `/info`, `/week`, `/schedule`, `/doc`
- Реєструє всіх, хто написав боту в особисті (`/start`) — у вкладку `requests`
- Якщо користувач заблокував бота — автоматично оновлює його статус у Google Sheet

---

## Як адмініструвати через Google Sheets

Відкрити Google Sheet можна командою `/doc` у особистому чаті з ботом (тільки для власників).

### Вкладка `groups` — групові чати

Тут зберігається список груп, в які бот надсилає оголошення.

| Колонка | Що це |
|---|---|
| `key` | Внутрішній ключ групи (`main`, `dev`). Використовується в `announcements.py` |
| `name` | Назва групи (для зручності, не використовується ботом) |
| `telegram_id` | ID чату в Telegram. Отримати можна командою `/chatid` в групі |
| `info` | Текст, який повертає команда `/info` в цій групі |
| `welcome` | Текст привітання (резерв на майбутнє) |

### Вкладка `users` — кому надсилати особисті повідомлення

Тут адмін вручну веде список людей, які отримують особисті розсилки.

| Колонка | Що це |
|---|---|
| `user_id` | Telegram ID користувача (число). Беремо з вкладки `requests` |
| `username` | Telegram username (без @) |
| `first_name` | Ім'я |
| `status` | `активний` — отримує розсилку. `заблокований` — заблокував бота |

**Щоб додати користувача до розсилки:**
1. Попроси людину написати боту в особисті (`/start`)
2. Відкрий вкладку `requests` — там з'явиться її запис
3. Скопіюй `user_id` і `username` у вкладку `users`
4. Постав статус `активний`

**Щоб вимкнути розсилку для користувача** — зміни статус на будь-яке інше значення або видали рядок.

### Вкладка `requests` — черга нових користувачів

Сюди бот автоматично записує всіх, хто написав `/start` в особисті. Адмін нічого тут не змінює — тільки читає, щоб підтягнути нових людей у `users`.

## Як додати нове оголошення

Відкрий файл `announcements.py` і додай новий рядок у список `ANNOUNCEMENTS`:

```python
{
    "text": "Текст повідомлення",
    "cron": "0 9 * * 1",       # коли надсилати (див. шпаргалку нижче)
    "chats": ["main"],          # в які групи: ["main"], ["dev"], ["all"]
    "users": ["all"],           # необов'язково: особисті повідомлення
}
```

**Поле `users`** — якщо потрібно надіслати в особисті:
- `["all"]` — всім активним користувачам з вкладки `users`
- `["username"]` або `["@username"]` — конкретному користувачу
- `["username1", "username2"]` — кільком
- якщо поле не вказано — особисті не надсилаються

**Динамічний текст** (обраховується в момент надсилання):
```python
"text": lambda: f"Сьогодні {get_day_of_week()} — гарного дня!"
```

**Cron-шпаргалка** (часовий пояс: Київ):
```
хвилина година * * день_тижня

0=пн, 1=вт, 2=ср, 3=чт, 4=пт, 5=сб, 6=нд

Приклади:
  25 8 * * 1-5   → 08:25 щодня пн-пт
  0 9 * * 1      → 09:00 щопонеділка
  0 17 * * 5     → 17:00 щоп'ятниці
  0 11 * * 1-5   → 11:00 щодня пн-пт
```

## Як внести зміни в код

Усі зміни вносяться через Git і GitHub. Після пушу в гілку `main` бот автоматично перезапускається.

```bash
git add .
git commit -m "опис змін"
git push origin main
```

Основні файли для змін:
- `announcements.py` — додати / змінити оголошення
- `handlers/commands.py` — додати нову команду
- `config.py` — змінити логіку роботи з Google Sheets

---

## Як запустити тести

**Локально:**
```bash
pytest tests/ -v
```

**На GitHub Actions** — тести запускаються автоматично при кожному пуші в `main`, до старту бота. Якщо тести не пройдуть — бот не запуститься.

Переглянути результат: GitHub → репозиторій → вкладка **Actions** → останній запуск → крок **Run tests**.

## Як перезапустити бота вручну

GitHub → репозиторій → вкладка **Actions** → зліва **TelegramAnnouncerBot** → кнопка **Run workflow**.

---

## Де дивитись логи

GitHub → репозиторій → вкладка **Actions** → останній запуск → крок **Run bot**.

Важливі повідомлення в логах:
- `Оголошення надіслано` — все добре
- `Особисте повідомлення надіслано` — особисте відправлено
- `заблокував бота` — користувач заблокував бота, статус оновлено автоматично
- `Помилка надсилання` — щось пішло не так, потрібно перевірити
